Acquires the color buffer ids attached to the frame. One frame contains several layers.

Users can only acquire part of the frame layers whose buffer contents need to update. After this function, users update the contens of the acquired texture buffers. This function is called just before user's rendering.

[in]frame_handleThe handle of frame object.
[in]swapchain_handlesAn array of swapchain handle indicate which swapchains to acquire.
[in]swapchian_countThe count of swapchain handles indicate how many swapchains to acquire.
[out]out_render_buffersThe acqiured render buffers of given swapchain handles.
Returns
The result of operation.

◆ NRFrameSetBufferViewport()

NRResult NRFrameSetBufferViewport ( NRHandle  rendering_handle,
NRHandle  frame_handle,
uint32_t  viewport_index,
NRHandle  viewport_handle 
)

Update an viewport element or append a new one at the end.

Don't insert any buffer viewport with the swap chain which will not display on screen. At least one buffer viewport with the swap chain should set to frame if you want it to display on screen. If viewport type is NR_VIEWPORT_PROJECTION, then need one buffer viewport for left display and one for right display. If viewport type is NR_VIEWPORT_QUAD, buffer content is shared between both left and right displays, two buffer viewports with the same swap chain are needed.

The viewport order in the frame decides the swap chain's order to display on screen. The 0th buffer viewport in the frame will display on bottom.

For display, SDK can guarantee the right result if less than four viewports for a single display. The result is not guaranteed if the count of viewports in the frame for single display is more than four.

Parameters
[in]rendering_handleThe handle of rendering object.
[in]frame_handleThe handle of frame object.
[in]viewport_indexIndex of the buffer viewport entry to update. If the viewport count is equal to the index, a new viewport entry will be added. The viewport count must not be less than the index value.
[in]viewport_handleA handle to the buffer viewport object.
Returns
The result of operation.

Set the color textures of frame object.

Developer can render the left texture and right texture image to two OpenglES GL_TEXTURE_2D objects seperately. Developer can also render both the left texture and right texture image to one OpenglES GL_TEXTURE_2D_ARRAY object by single-pass.

Note: Developers can convert from the GLuint to void* by type casting to get left_display_texture and right_display_texture parameters. for example: GLuint left_display; // render left_display GLuint right_display; // render right_display void *color_textures[2]; color_textures[0] = (void *) left_display; color_textures[1] = (void *) right_display; // call NRFrameSetColorTextures

Parameters
[in]rendering_handleThe handle of rendering object.
[in]frame_handleThe handle of frame object.
[in]color_texturesThe pointer of color texture array.
[in]color_texture_countThe count of color textures in the texture array.
Returns
The result of operation.

◆ NRFrameSetFocusPlaneNormal()

NRResult NRFrameSetFocusPlaneNormal ( NRHandle  rendering_handle,
NRHandle  frame_handle,
const NRVector3f *  plane_normal 
)

Set the normal of fucus plane.

Note: Developers set the focus plane info of the frame, and the rendering system will use the fucus plane to do plane based reprojection. Normally the normal is best set to be the opposite of the main camera's forward vector(0.0, 0.0, 1.0). If the content is actually all on a plane (like text), set the normal to the normal of the plane and ensure the user does not pass through the plane. Make sure the normal is in right-hand coordinate system.

Parameters
[in]rendering_handleThe handle of rendering object.
[in]frame_handleThe handle of frame object.
[in]plane_normalThe view space normal of plane. If the point is the cross point with the main camera's forward ray, normal always set to be (0.0, 0.0, 1.0)
Returns
The result of operation.
Deprecated:
Only for compatible

Subumits the frame to SDK. SDK will composite all the frame buffers and do rerojection as well as distortion afterwards.

Call this function after setting the frame info like buffer viewport, pose, present time and fucus plane etc. Parameter frame_handle is invalid after calling this function, do not use it anymore. Make sure to call this function on rendering thread.
