◆ NRBufferViewportCreate()

NRResult NRBufferViewportCreate ( NRHandle  rendering_handle,
NRHandle *  out_viewport_handle 
)

Creates the buffer viewport object. Buffer viewport tells SDK where to display the given layer buffer on the screen.

Call NRBufferViewportDestroy to release the memory if no longer used. If the viewport's source layer is 3d buffer rendered by camera, then call NRBufferViewportSetSourceFov to set the camera's fov as the source fov, call NRBufferViewportSetTransform to set the position of the camera. The 2d layer buffer read from android surface or just a staic picture etc., should be rendered on a plane in the 3d space. If the viewport's source layer is 2d buffer, call NRBufferViewportSetTransform to set a matrix to transform the plane from local space to reference space.

Parameters
[in]rendering_handleThe handle of rendering object.
[out]out_viewport_handleThe handle of the newly created buffer viewport object.
Returns
The result of operation.

Gets the transform that positions the viewport in reference space.

NR_REFERENCE_SPACE_VIEW means the head space.

For projection viewport, transform transforms the target component from local space(or object space) to reference space. For quad viewport, transform transforms the points of quad from local space(or object space) to reference space. The out_transform is in right-hand coordinate system.

◆ NRBufferViewportSetFocusPlane()

NRResult NRBufferViewportSetFocusPlane ( NRHandle  rendering_handle,
NRHandle  viewport_handle,
NRReferenceSpaceType  reference_space,
const NRVector3f *  plane_point,
const NRVector3f *  plane_normal 
)

Sets the viewport's focus plane.

Developers set the focus plane info of the viewport, and the rendering system will use the fucus plane to do plane based reprojection.

NR_REFERENCE_SPACE_VIEW means the head space.

Normally the normal is best set to be the opposite of the head's forward vector(0.0, 0.0, 1.0) in view space. If the content is actually all on a plane (like text), set the normal to the normal of the plane and ensure the user does not pass through the plane. Make sure the point and normal are in right-hand coordinate system.

Note: Focus plane can nly be set for NR_VIEWPORT_PROJECTION viewport, sdk would ignore fucus plane set for NR_VIEWPORT_QUAD viewport.

Parameters
[in]rendering_handleThe handle of rendering object.
[in]viewport_handleThe handle of viewport object.
[in]reference_spaceThe reference space of plane.
[in]plane_pointThe point on plane or cross point with the head's forward ray in reference space.
[in]plane_normalThe normal of plane in reference space. If the point is the cross point with the head's forward ray, normal always set to be (0.0, 0.0, 1.0) in NR_REFERENCE_SPACE_VIEW space.
Returns
The result of operation.

◆ NRBufferViewportSetTransform()

NRResult NRBufferViewportSetTransform ( NRHandle  rendering_handle,
NRHandle  viewport_handle,
NRReferenceSpaceType  reference_space,
const NRTransform *  transform 
)

Sets the transform that positions the viewport in reference space.

If the viewport is world-locked, then reference_space should be NR_REFERENCE_SPACE_GLOBAL. If the viewport is head-locked, then reference_space should be NR_REFERENCE_SPACE_VIEW and the transform should have no relation with head movement. NR_REFERENCE_SPACE_VIEW means the head space.

For projection viewport, transform transforms the target component from local space(or object space) to reference space. For quad viewport, transform transforms the points of quad from local space(or object space) to reference space. Make sure the transform is in right-hand coordinate system.

The vertices of quad in local space(or object space) before sacle are
(-0.5, -0.5, 0), (0.5, -0.5, 0), (-0.5, 0.5, 0), (0.5, 0.5, 0).

Parameters
[in]rendering_handleThe handle of rendering object.
[in]viewport_handleThe handle of buffer viewport object.
[in]reference_spaceThe reference space of transform.
[in]transformTransforms a quad/camera from local space(or object space) to reference space.
Returns
The result of operation.
