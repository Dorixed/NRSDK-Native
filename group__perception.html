 NR_ENUM (NRTrackingReason)
 Tracking reasons indicated validity of the current head pose. NR_TRACKING_REASON_NULL: Tracking is working properly. NR_TRACKING_REASON_INITIALIZING: Tracking is initializing. It will complete in a short time. Please do not use this state. NR_TRACKING_REASON_EXCESSIVE_MOTION: Tracking lost due to excessive motion. Please ask the user to move the HMD slowly. NR_TRACKING_REASON_INSUFFICIENT_FEATURES: Tracking lost due to insufficient features. Please ask the user to move to a different area with surfaces which have more details. NR_TRACKING_REASON_RELOCALIZING: Tracking is relocalizing. It will complete in a short time. Please do not use this state. NR_TRACKING_REASON_LOW_LIGHT: Tracking lost due to low light. Please check the environment light.

 NR_ENUM (NRTrackingState)
 Tracking state indicated the state of the trackable object. It is currently used with trackable plane and trackable image. NR_TRACKING_STATE_TRACKING: The object is currently tracked and the pose is updated. NR_TRACKING_STATE_PAUSED: The SDK has paused tracking the object and the pose is inaccurate with great probability. This may happen when the tracking is paused or lost. NR_TRACKING_STATE_STOPPED: The SDK has stopped tracking the object and will not re-track it.
